# Hacker Bob v1.1.8 Release Notes

## Highlights

`v1.1.8` is a documentation and release-hygiene update for first-time installs, especially when a user gives this repository URL to another AI coding agent.

The README now puts the AI-agent install flow first, makes the human install path explicit, and explains that `bountyagent` is the expected internal MCP server name. The user-facing commands remain `/bob-hunt`, `/bob-status`, `/bob-debug`, `/bob-update`, and `/bob-egress`.

## Changes

- Moved AI-agent installation instructions before the human quickstart.
- Clarified that the source checkout is normally the installer, while the target Claude Code project is where Bob should be installed and run.
- Added a direct MCP load check with `node -e "require('./mcp/server.js'); console.log('MCP ok')"` to the repo-link install flow.
- Documented the stale-runtime failure mode where `mcp/lib/tool-registry.js` exists but `mcp/lib/tools/index.js` is missing, producing `Cannot find module './tools/index.js'`.
- Added ignore rules for local `.claude/bob/` install metadata created when testing installs from the source checkout.

## Install

```bash
npx -y hacker-bob-cc@1.1.8 install /path/to/your/project
```

After installing, fully restart Claude Code in that project and verify `claude mcp list` shows `bountyagent` connected.
