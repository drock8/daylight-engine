# Hacker Bob v1.3.5 Release Notes

`v1.3.5` hardens the shared MCP runtime contracts and release gates used by Claude, Codex, and generic MCP installs.

## What Changed

- Extracted implicit runtime behavior into explicit contracts and stores for findings, session state, verification rounds, wave handoffs, grade verdicts, pipeline events, pipeline session artifacts, handoff signing, and session authority.
- Rewired large runtime owner modules around those contracts while preserving their public MCP tool behavior.
- Centralized smart-contract egress policy and direct public HTTPS enforcement across chain clients, source fetches, and harness handoff paths.
- Tightened transport, tool registry, telemetry, session read/write guards, and secret screening around the shared authority model.
- Added deterministic package/release checks: authority inventory freshness, clean release audit, package surface documentation, dependency freshness checks, and stricter package policy tests.
- Regenerated Claude, Codex, generic MCP, role prompt, skill, and documentation surfaces to reflect the shared runtime behavior.
- Clarified Claude statusline update hints and suppress stale cached downgrade prompts after local installs.
- Removed obsolete replay/benchmark scripts and the stale generated offline-guide PDF from source control.

## Review-cycle hardening

The first review cycle on this branch surfaced four P1/HIGH blockers and a MEDIUM regression. They are all fixed in this release:

- **Null at dispatch.** `validateObject` in `mcp/lib/tool-validation.js` previously rejected explicit `null` for required + nullable fields. Required + `type: ["string", "null"]` now means "key must be present, value may be null", which is what the four affected tool fields (`bounty_write_verification_round.notes`, `bounty_write_evidence_packs.packs[].redaction_notes`, `bounty_write_grade_verdict.feedback`, `bounty_write_grade_verdict.findings[].feedback`) actually need.
- **Session backfill.** `mcp/lib/session-authority.js` now runs `normalizeSessionStateDocument` before `assertLegacyFailClosedFields`, and the fail-closed list is trimmed to scope identity (`target`, `target_url`). v1.3.4-shaped sessions on disk read cleanly with normalizer defaults instead of failing with `STATE_CONFLICT`.
- **Handoff provenance flag.** New per-session `handoff_provenance_required` flag in session state. v1.3.5+ sessions are initialised with the flag set, which makes `validateHandoffToken` and `validateHandoffProvenance` reject the previous `legacy_unverified` downgrade with a clear re-init hint. Legacy sessions resumed from disk default to `false` so in-flight pre-v1.3.5 work can drain. v1.3.6 will drop the legacy path entirely (see Unreleased in [CHANGELOG.md](../../CHANGELOG.md)).
- **Unified wave readiness.** `buildWaveReadiness` in `mcp/lib/wave-handoff-store.js` now validates handoff content and provenance inline when given a domain, exposes `invalid_agents`, and gates `is_complete` on `missing.length === 0 && invalid.length === 0`. `bounty_apply_wave_merge` surfaces invalid agents in `missing_surface_ids` so the orchestrator cannot quietly advance past tampered handoffs.
- **Chain-notes resilience.** `summarizeStructuredHandoffChainNotes` in `mcp/lib/pipeline-session-artifacts.js` distinguishes ENOENT on the assignment file (legitimate legacy state) from validation failure (tampering). The ENOENT path falls back to direct `chain_notes` parsing capped at 10 per session and reports `unsigned_handoff_count`; validation failure still strict-drops.
- **Deprecation shim.** `mcp/lib/pipeline-analytics.js` re-exports four pipeline-event helpers that moved to `pipeline-events.js`, so existing importers do not break.
- **CI dependency freshness.** `.github/workflows/release.yml` now runs `npm run release:check:dependencies` as its own step on every tag build.
- **Installed policy-replay SDK resolution.** `testing/policy-replay/replay.mjs` now bare-imports `@anthropic-ai/claude-agent-sdk` first and falls back to `createRequire(<workspace>/mcp/server.js).resolve(...)` when the workspace layout puts the SDK under `mcp/node_modules/`. `scripts/install.js` `copyRuntimeNodeDependencies` also walks `optionalDependencies` (required-wins-over-optional semantics) so the SDK's current-platform native package propagates when npm installed it in source. `test/install-smoke.test.js` asserts the SDK resolves from `mcp/server.js` after install, and that any current-platform native package present in source is copied to `<workspace>/mcp/node_modules`. End-to-end `query()` invocation is tracked as a v1.3.6 follow-up (see [CHANGELOG.md](../../CHANGELOG.md) Unreleased).

### Pre-v1.3.5 sessions

The `handoff_provenance_required` flag is set when a session is initialised on v1.3.5+. Sessions that were started on v1.3.4 and resumed on v1.3.5 inherit the legacy default (`false`) so they can drain to completion. To take advantage of the strict provenance rejection on a session in flight, re-init with `bob init <target>`. v1.3.6 will require this for every session.
